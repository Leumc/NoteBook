类的基本注意事项：

  1. 类的成员可以是变量，也可以是函数。
  2. 类的成员变量也叫属性。
  3. 类的成员函数也叫方法/行为，类的成员函数可以定义在类的外面（也可以在不同的文件中）。
  4. 用类定义一个类的变量叫做创建（或实例化）一个对象。
  5. 类的成员变量和成员函数的作用域和生命周期与对象的作用域和生命周期一致。

类的访问权限：

类的访问权限有三种，分别为public、private和protected，它们的访问权限有如下：

  1. public：公有的成员可以在被外部访问。
  2. private：私有的成员无法被外部访问，只有类的内部可以访问，这个访问权限不会被继承给派生类。
  3. protected：受保护的成员无法被外部访问，只能在类及其派生类的内部访问，

在默认情况下（不手动设置访问权限），类的成员均被初始化为private，而在结构体中，其成员初始为public。

设置访问权限的方法：

在类中，使用权限关键字+冒号的方式来设置访问权限，在该标识之下的所有成员，其访问权限都会被重设为指定的类型，例如：

类的简单使用：

  1. 类的成员可以是任意数据类型（包括类、指针、结构体、枚举）
  2. 在C++11标准中，可以为类的成员指定缺省值。
  3. 使用new动态创建对象，使用delete释放对象。
  4. 不建议在外部读写类的属性（即不建议将类的属性声明为public），而应该实现用于读写属性的方法，来间接访问。
  5. 如果需要一个只是纯粹的数据的自定义类型，建议使用结构体，而当需要实现对自定义类型自身的特殊操作时，建议使用类。

注意： 如果一个成员函数在类中被实现，其将自动成为内联函数，而在类的外部用::解析符定义的成员函数，则会是一个普通函数（也可加上inline限定符来使之成为内联函数）。

构造函数和析构函数：

  • 构造函数： 在创建对象时，自动调用构造函数来对该对象进行初始化。
  • 析构函数： 在对象被销毁前，自动调用析构函数完成该对象的清理操作。（当类中用到动态分配内存时，需要自定义释放方法）。

1. 构造函数：

编写构造函数的语法如下：

1
类名(参数列表){初始化语句}//不需要写返回值类型，也没有返回值

没有参数的构造函数也叫做默认构造函数。

注意：

  1. 构造函数需要在外部调用，其访问权限必须是public
  2. 构造函数名需要自动化执行，其函数名必须与类名完全一致。
  3. 不能手动调用，只会在创建对象时被自动调用一次。

2. 析构函数

编写析构函数的语法如下：

1
~类名(参数列表){释放语句}//同样的，也没有返回值，不需要写返回值类型

注意：

  1. 与构造函数一样，访问权限也必须是public，函数名也必须与类名完全一致。
  2. 前面的~是区分构造函数与析构函数的标志，不能省略。
  3. 析构函数没有参数，不能重载。
  4. 析构函数在对象被销毁（delete或程序终止时的自动资源释放）的时候会被自动调用一次，但析构函数也可以使用类名.~类名()来手动调用。

构造函数和析构函数的细节：

  1. 如果没有为类提供构造函数或析构函数，那么编译器会自动提供一个空的默认构造函数或析构函数；而如果提供了，那么编译器将不会为其提供空的构造函数。在提供了析构函数的情况下，直接用类名 对象名;的不带括号的创建方式将会出现”没有合适的默认构造函数可用“的提示。
  2. 如果类有无参数的构造函数，在创建对象时，不要在对象后面加上括号，而应使用类名 对象名;的方法来创建。 因为对于类名 对象名();这个语句，编译器会将其解释为声明一个没有参数、返回值类型为该类函数。
  3. 类名();的意思不是调用该类的构造函数，而是创建了一个该类的匿名对象/临时对象，这个对象没有名字，在运行完其所在语句时，将会被立刻删除。
  4. 使用new创建对象时，会调用匹配的构造函数，其返回的是指向新创建的对象的指针；使用delete销毁对象时，会调用析构函数。
  5. 不建议在构造和析构函数中写太多代码（因为没有返回值，无法只到运行的结果），如果有比较复杂的初始化/释放逻辑，可以额外创建一个方法来完成。
  6. 在C++11标准中，允许使用统一初始化列表来创建函数，例如类名 对象名 = {参数一,参数二}等价于类名 对象名(参数一,参数二)。
  7. 如果类的成员也是类，创建对象的时候，先构造成员类；销毁对象时，先析构成员类。

拷贝对象：

如果已经有了一个对象，想要复制一个，那么可以使用拷贝操作。拷贝操作有两种形式：

  1. 类名 新对象名(已存在的对象名)：调用了拷贝构造函数
  2. 类名 新对象名 = 已存在的对象名：直接把原对象赋值给了新对象

拷贝构造函数：

默认情况下，如果没有提供默认的拷贝构造函数，那么编译器会提供一个默认的拷贝构造函数，它的作用是将已存在对象的成员变量原封不动地复制给新对象。

编写拷贝构造函数的语法如下：

1
类名(const 类名& 对象参数名,其他参数列表){拷贝操作}//实际上就是一个重载的构造函数

没有其它参数列表的拷贝构造函数被称为默认拷贝构造函数，无论如何，都会有一个默认拷贝构造函数在类中（无论是自己定义的还是编译器提供的，重载的拷贝构造函数的定义不会影响编译器提供默认拷贝构造函数）。 ^03b312

注意：

  • 【使用MSVC编译器的情况下】如果一个函数的参数为是以值传递的类对象，那么在复制参数时，会调用拷贝构造函数来进行复制操作。所以说，如果拷贝构造函数里夹带了私货，那么在传值是实际起作用的是被塞了私货的对象。 【使用g++编译器的情况下】同样进行上述操作时，g++认为不需要复制一个新的对象，直接换一个变量名继续用即可，两者的地址是完全一致的，相当于引用。
  • 其他注意事项与常规构造函数一致。

浅拷贝与深拷贝：

假设类的属性中有指针变量，那么在拷贝时如果直接将已存在对象的所有属性都复制给新对象，那么就会导致这两个对象中的这一指针变量指向同一片内存区域，一个对象对这个区域的修改将会影响到另一个对象。

更加危险的是，如果在析构函数中释放了这个指针，那么当一个对象释放时，该指针会被释放掉，另一个对象的该指针就变成了野指针，对这个野指针进行释放，会直接导致程序发生致命错误而崩溃！

以上的问题被称为浅拷贝。

深拷贝：

要解决这个问题，做法非常简单，不要使用编译器提供的默认拷贝构造函数，使用自定义的拷贝构造函数来进行特殊拷贝。

  • 对于普通的数据，直接复制即可。
  • 对于指针或者引用变量这样特殊的数据，可以考虑使用memcpy(目标指针,源指针,内存大小)来复制。

这样的操作被称为深拷贝。

初始化列表：

前面提到了一个用于创建对象的统一初始化列表，哪个是C++11新增的，而此处的初始化列表是C++最初就有的，两者不是一个概念。

初始化列表的语法如下：

1
2
变量名(参数列表):成员1(初始值1),成员二(初始值2),……
//初始值可以是常量，也可以是形参或者表达式

在调用该函数的时候，会将成员初始化为括号内指定的值。

为什么要使用初始化列表？

  1. 对于类对象参数来说，如果以值传递且初始化操作放在函数体内，那么会调用类的普通构造函数创建一个实参再用这个实参进行初始化，这是两步操作；而如果使用初始化列表进行初始化，那么将会调用类的拷贝构造函数直接将传递的类对象拷贝给成员，这是一步操作。因此在初始化类的情况下效率更高。
  2. 如果类的属性是常量或者引用，由于常量和引用必须在创建时初始化，所以在函数体内初始化的操作由于创建和赋值分离会有“必须在创建时初始化”的报错，而初始化列表由于是一步完成的，所以能够对常量和引用进行初始化。因此在初始化常量或引用的情况下，必须使用初始化列表。

注意：

  • 如果初始化的类没有默认构造函数（即没有参数的构造函数），那么会由于无法创建空对象而报错，更别说赋值了。在这种情况下，必须使用初始化列表，因为所有类都一定会有拷贝构造函数。

const与mutable：

const修饰的成员函数不能修改非mutable的成员变量，使用语法如下：

1
2
3
mutable 变量类型名 变量名;//这是一个mutable修饰的成员变量
返回值类型名 函数名() const {函数体};
//这是一个const修饰的成员函数，它只能对mutable修饰的变量进行更改

同时，const修饰符还会影响访问权限：const方法和const修饰的类对象只能访问类中的const方法（属性可以正常访问，但不能修改），而不能访问其非const方法。

注意：构造和析构函数不能被const修饰。

this指针：

this是一个指向对象本身的指针，在调用类方法时，this会作为隐藏参数传递给类方法，对其解指针即可得到对象本身。

与C语言一样，static修饰的类成员将被储存在静态储存区中，即便其定义在类中，也可以在全局通过作用域解析运算符来访问，这个操作不需要创建对象，静态的成员在整个程序间共享，例如：

class Example{
public://调整为公有来外部访问
	static int a;
	static int getValue(){
		return a;
	}
};

int Example::a=9;//静态成员必须在全局作用域中进行初始化

int main(){
	cout<<Example::getValue()<<endl;//输出9，不创建对象也可以使用
	Example::a=6;//将其改为6
	cout<<Example::getValue()<<endl;//输出6，a被全局改变
	Example test;
	cout<<test.getValue()<<endl;//也会输出6（值被全局变更），对象也可以访问
}

注意：

  1. 静态的成员与类独立，且在全局中只有一份。
  2. 静态成员函数只能访问静态成员，不能访问非静态成员。
  3. 静态成员函数没有this指针，静态成员函数不属于某一特定对象。
  4. 私有静态成员无法在外部访问（仍然可以在全局作用域中初始化）。

类的私有成员无法被外部访问，但我们可以主动为某些函数或类开后门，让它们称为友元，从而获得对私有成员的访问权限。

class Example{

	}

	Example eg;
	cout<<eg.a;//没有权限，不能访问私有成员

}

	return eg.a;
}

int main(){
	Example eg;
	Other ot;
	cout<<ot.getValue(eg)<<endl;
}

槽点：

  1. 因为友元函数访问视其为友元类必须传入一个类对象的引用，所以需要这个类在友元函数所在类之前定义，而要声明该函数是一个类的友元函数，需要这个友元函数所在类在这个类之前定义？所以既要满足包含友元函数的Other类在Example类之前定义（为了进行友元声明），又要满足Example类在Other类之前定义（为了传入Example对象的引用）？ ->使用前向声明，令Other知晓Example的存在
  2. 因为友元函数的声明必须在友元函数实现之前，所以在能满足上面条件的情况下，还要让getValue方法在Example之下实现？ ->将getValue函数的实现分离出来，放到Example的下面

声明友元成员函数的步骤：

  1. 将声明所在类进行前向声明。
  2. 将友元成员函数所在类的定义放在前向声明和声明所在类的定义之间。
  3. 将友元成员函数的实现从类中分离，放到声明所在类下方。
  4. 在声明所在类中写下友元声明语句（包含作用域解析符）。

注意事项：

  1. 友元关系无法被继承。
  2. 友元关系是单向的（我不是我友元的友元）

自动类型转换：

类的构造函数还可以用作类型转换，如果一个构造函数只有一个参数，那么它可以被视为转换函数，当尝试将与参数类型相同的量赋给类对象的时候，会自动调用这个构造函数来创建对象，例如：

class Example{
	int a;
public:
	Example(int a){
		this->a=a;
	}
};

int main(){
	Example test=6;//等价于Example test(6);
	Example test=6.6;
	//先将double的6.6转换成int的6（截断），再调用构造函数
	//等价于Example test((int)6.6)
}

感觉跟C++11的统一初始化列表差不多，Example test=6完全可以理解为Example test={6}，即将一个只有一个值的列表作为了统一初始化列表。

程序在任何类型不一致的时候下都会尝试触发这种自动类型转化，例如一个函数返回值类型为上面的Example但实际返回了6，那么就会触发自动类型转换。

如果希望一个构造函数不要被用于自动类型转换，可以为其加上explicit（意为精确的）修饰符，加上explicit修饰符的构造函数将不会被尝试用作自动类型转化，但如果使用强制类型转换，即便是explicit的构造函数也会被调用（例如Example test=(Example)6;）。
