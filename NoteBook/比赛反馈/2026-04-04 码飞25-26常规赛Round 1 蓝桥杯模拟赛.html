T745064 试题 B ：旅馆

当然，这道题我没做出来，但还是说一下我的一点思路：

因为$|A_i-i|\le1$，所以对于第$i$个分店，只能有$i-1,i,i+1$这三个值可以选择，而对于第1个分店，实际上只有两种选择，即$i,i+1$：

  • 如果第一个分店选择了$i$的话，那么下一个分店的$i-1$就无法选择，又演变成第一个分店的情况；
  • 如果第一个分店选择了$i+1$的话，那么下一个分店则必须选择$i-1$，否则由于$|A_i-i|\le1$的规则，这个$i-1$将无处安放。

这个思路有问题吗？没有一点问题！那么问题出在了哪呢？出在了我的高中数学上……

我的想法是，只有选择$i$的时候，下一个分店才会有选择权，所以如果上一个分店选择了$i$，那么下一个分店有两种选择，此时方案数乘2，并尝试两种选择，写个递归就行了。至于独立性以及分类加法什么的都抛诸脑后，我的眼里只剩下了乘法～

官方题解的解法是用了一点点简单的动态规划思想：

定义$dp[i]$表示只考虑前$i$个分店的方案数，那么排完第$i$个分店的方案数是怎么来的呢（从来源角度考虑）？方案数的来源有以下两种：

  1. 第$i-1$个分店选择了自己（即分店编号等于客流量），来源是第$i-1$个分店
  2. 第$i-1$个分店由于第$i-2$个分店没有选择自己而必须选择前一个分店编号的客流量，来源是第$i-2$个分店（第$i-1$个分店别无选择）

所以状态转移方程就是 $$dp[i]=dp[i-1]+dp[i-2],dp[1]=1,dp[2]=2$$ 是的，这是一个斐波那契数列，我们要求的就是斐波那契数列的第2026项：

#include <bits/stdc++.h>
using namespace std;
int dp[2030];
int main(){
	dp[1]=1;dp[2]=2;
	for(int i=3;i<=2026;i++){
		dp[i]=(dp[i-1]+dp[i-2])%1000000007;
	}
	cout<<dp[2026];
	//结果是749347953
}

#include <bits/stdc++.h>
using namespace std;
int n,m;
int ns[505][505];
int begin_r=-1,begin_c=-1,target_r=-1,target_c=-1;
bool vis[505][505];
int ds[4][2]={{-1,0},{0,-1},{1,0},{0,1}};
int ans=-1;
bool found;
void dfs(int r,int c,int depth){
	if(found)return;//如果找到了，就直接走人
	vis[r][c]=true;
	if(r==target_r&&c==target_c){
		found=true;
		ans=depth;
	}
	for(int i=0;i<4;i++){
		int nr=r+ds[i][0],nc=c+ds[i][1];
		if(vis[nr][nc])continue;
		if(ns[nr][nc]==-1||ns[nr][nc]>=ns[r][c])continue;
		dfs(nr,nc,depth+1);
	}
}
int main(){
	cin>>n>>m;
	for(int i=1;i<=n;i++){
		string s;cin>>s;
		for(int j=1;j<=m;j++){
			char c=s[j-1];
			if(c=='#')ns[i][j]=-1;
			else if(c=='S'){
				begin_r=i;
				begin_c=j;
			}
			else if(c=='T'){
				target_r=i;
				target_c=j;
			}
		}
	}
	for(int i=1;i<=n;i++){
		for(int j=1;j<=m;j++){
			if(ns[i][j]!=-1){
				ns[i][j]=abs(i-target_r)+abs(j-target_c);
			}
		}
	}
	dfs(begin_r,begin_c,0);
	cout<<ans;
}
