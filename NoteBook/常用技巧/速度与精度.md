在刷题时，有时候会遇到内置操作的速度太慢或者精度太低无法AC的情况，这时就需要我们自行造一个比官方更快的轮子，本篇文章就记录一下鄙人了解过的部分快速/高精操作的模板：
# 我们需要更快一点：

### 快速读取：

如果你是一个习惯使用`cin`的Coder，或者想要`scanf`的速度又想要`cin`的简洁语法，你只需要对`cin`进行一些微调：

```cpp
ios::sync_with_stdio(false);//关闭标准流同步，此时不能scanf/printf和cin/cout混用
cin.tie(nullptr);//cin与cout解绑
```

如果还是不够快，那就需要自己造一下轮子了。假设输入数据是一大堆的整数的话，我们可以用这么一个快读函数来彻底消除格式解析的时间开支：

```cpp
inline int read(){
	char c=getchar();int x=0,f=1;//x是最终的值，f是符号表示
	for(;!isdigit(c);c=getchar())if(c=='-')f=-1;//寻找第一个有效的数字或符号位置
	for(;isdigit(c);c=getchar())x=x*10+c-48;//-48转换字符为一般整数
	return x*f;//返回结果
}
```
### 快速写入：

实际上应该叫作快速输出，主要是跟上面是一套的，这个名字更加常用。首先第一中自然是`ios`解绑：

```cpp
ios::sync_with_stdio(false);//关标准流同步
cout.tie(nullptr);//cout与cin解绑
```

另外一种更快的方法依然是自己造轮子：

```cpp
inline void write(int x){
    if(x<0){putchar('-');x=-x;}
    else if(x>9)write(x/10);
    putchar(x%10+48);
}
```
### 快速幂：

幂运算的时间复杂度还是太高了，如果不得不用幂运算，还是建议不要直接`a^b`，可以使用一个非常简单的快速幂函数来算。

举个例子，假设我们要算的是$7^{11}$，那么正常算法肯定是11个7乘起来，那我用一下乘法结合律，写成$7^8\times 7^2\times 7^1$行不行？当然行，没毛病。
那么这样写有什么好处？这样写的话每一个因子都能在$\{7,7^2,(7^2)^2,((7^2)^2)^2\cdots\}$这个不断自我平方的递推序列中得到，所以时间复杂度是$O(log_2N)$的，这里的$N$是指数。
那么为什么是这几个因子呢？ 如果把11写成二进制的话就是`0b1011`，不难发现，如果对应数位上是1，就把当前的递推值乘到结果里，否则就不乘。

```cpp
int quickPower(int a,int b,int m){
	int ans=1;int base=a;//ans是结果，初始化为1,base是当前的递推值，初始为底数
	while(b>0){
		if(b&1)ans=ans*base%m;//如果指数的最后一位是1,那么这个因子需要乘
		base=base*base%m;//递推下一个递推值
		b>>=1;//b右移一位
	}
}
```

不要取模就把`m`删了，不过大概率是要的。
# 这么低的精度能用？

### 高精乘法：

C++不是Python，精度最高的无符号长整型存储的最大值是$O(2^{64}-1)$，有时候题目要我们算的值就是会超，好像也没有什么办法。于是就需要一种新的存储方式（假设需要500位精度）：

```cpp
void multiply(int a[],int b[],int res[]){//计算a*b并保存到res
	int temp[1000];
	for(int i=0;i<500;i++){
		for(int j=0;j<500;j++){
			temp[i+j]+=a[i]*b[j];
		}
	}
	int t=0;
	for(int i=0;i<500;i++){//如果不希望截断就改成1000（两倍）
		temp[i]+=t;
		res[i]=temp[i]%10;
		t=temp[i]/10;
	}
}
```

实际上就是模拟了一下竖式运算，精度可调，根据这个思路（先都加起来，最后一起进位），也能写出高精加法。
