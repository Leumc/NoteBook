# 问题分析：

Em……这道题说每次新增一个人的成绩，就更新一次排名，也就是说如果用暴力算法每次都排一次序的话总共会有$O(n^2)$的时间复杂度，TLE绝对的。

然后呢，我了解到有一种十分适合动态维护第k大/小的元素的数据结构——对顶堆，于是就写一下吧。
# 对顶堆的实现：

首先对顶堆是堆，但堆还是太抽象了，我们使用堆实现的优先队列来实现。优先队列（`priority_queue`）的特性是它保证对首一定是这个队列所有元素的最大/最小值（不是全部排序，所以快）。

我们需要创建两个优先队列，一个最大优先队列，以及一个最小优先队列：

```cpp
priority_queue<int> max_hp;//默认是最大优先队列，队头是最大值
priority_queue<int,vector<int>,greater<int>> min_hp;
//指定为最小优先队列，队头是最小值
```

这两个队列的队头对在一起能够表示序列中的所有元素，所以我们只要保证前k大的元素都在`min_hp`中，那么`min_hp.top()`表示的就是那个获奖的基准分数线。

所以我们只需要确保：
1. 比`min_hp.top()`大的元素应该推入`min_hp`中
2. `min_hp`的`size`应该始终等于`k`，即$\max(1,[p \times w\%])$

写一个动态判断应该放在哪个队列里的`push`函数：

```cpp
void push(int x){
    if(min_hp.empty())min_hp.push(x);
    //k最小是1，所以min_hp中总应该有一个元素
    if(x>min_hp.top())min_hp.push(x);
    else max_hp.push();
    //动态判断应该放在哪个队列里，等于随便，反正接下来要进行长度限定
    if(min_hp.size()>k){
        max_hp.push(min_hp.top());
        min_hp.pop();
        //多了，就把min_hp中的最小值转移到max_hp
    }
    else if(max_hp.size()<k){
        min_hp.push(max_hp.top());
        max_hp.pop();
        //少了，就把max_hp中的最大值转移到mim_hp
    }
    //如果正好的话就不做任何操作
}
```

接下来补充一下输入输出以及k的更新即可。
# 完整代码：

```cpp
int n,w,p,k;
priority_queue<int> max_hp;
priority_queue<int,vector<int>,greater<int>> min_hp;
void push(int x){
    if(min_hp.empty())min_hp.push(x);
    if(x>min_hp.top())min_hp.push(x);
    else max_hp.push(x);
    if(min_hp.size()>k){
        max_hp.push(min_hp.top());
        min_hp.pop();
    }
    else if(max_hp.size()<k){
        min_hp.push(max_hp.top());
        max_hp.pop();
    }
}
//快读再提一下速
int read(){
    char c=getchar();int x=0,f=1;
    for(;!isdigit(c);c=getchar())if(c=='-')f=-1;
    for(;isdigit(c);c=getchar())x=x*10+c-48;
    return x*f;
}
int main(){
    n=read();w=read();
    for(int i=1;i<=n;i++){
        p++;
        k=max(1,p*w/100);
        push(read());
        cout<<min_hp.top()<<" ";
    }
}
```
