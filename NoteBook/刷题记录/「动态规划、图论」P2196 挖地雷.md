# 问题分析：

这是一道动态规划题，最重要的就是设好状态。我们需要求的是能挖到的最大地雷数，并且还要求输出挖到最大地雷数的路径。我们先不管路径，先想怎么把最大地雷数给找出来。
### 寻找所有路径中最大的地雷数：

首先，所有地窖之间并不是都相连的，输入数据中给出了地窖之间的相连关系。这很容易让我们想到图论，所以我们可以根据输入数据建立一个单向的邻接表（因为题目并不允许我们往回走），用`cs[i]`表示能连通到第`i`个地窖的所有地窖的序号。

同时，题目还说，我们可以从任意一个地窖开始走。我们先不管这个，先想一下状态怎么设（假设从第一个地窖开始走），我们可以很容易地想到，可以设$dp_i$表示到达第`i`个地窖时挖到的最大的地雷数。这样我们的初始状态和状态转移方程是（$as_i$表示第`i`个地窖中有几颗雷）：$$dp_1=as_1,dp_i=max\{dp_{cs[i][0]},dp_{cs[i][1]},\cdots,dp_{cs[i][n]}\}+as_i$$那么如果我们要找到从第二个地窖开始的最大地雷数，只要把初始化条件改成$dp_2=as_2$即可，后面的转移方程无论如何都是要跑一遍的。既然如此，如果我们要同时考虑从第一个地窖和从第二个地窖的情况，是不是把两个初始化条件都执行一遍就行了？

既然有了思路，那么就验证一下，假设$as=\{1,200,4,2\}$，连接关系为$1\rightarrow3\rightarrow4,2\rightarrow4$，模拟一下转移过程：

$i=1:$$dp_1$初始化为$as_1=1$，没有地窖与1号地窖连通，$dp_1=1;$
$i=2:$$dp_2$初始化为$as_2=200$，没有地窖与2号地窖连通，$dp_2=200;$
$i=3:$地窖1与地窖3连通，$dp_3$更新为$dp_1+as_3=5;$
$i=4:$地窖2和地窖3都与地窖4连通，$dp_2=200,dp_3=5$，取最大值，$dp_4=dp2+as_4=202;$

我们可以发现，当程序意识到从2来的这条路径比从1来的这条路径更优时，最终结果转移到了第二条路径上，所以这样的方法是正确的。推而广之，只要我们令$dp_i=as_i$，状态转移之后就能得到最优的路径。
### 记录路径：

首先思考一下路径确定是在哪里发生的。很明显是在状态转移的时候，而我们实际上并不需要记录下整条路径，我们只需要记录下每个地窖在最优的时候，是从哪里来的就行。

设$qs_i$为最优路径下，到达第`i`个地窖的上一个地窖编号。而最开始的地窖$qs=0$，我们只要一路溯源上去，只要无法溯源，就能找到最优的路径。（剩下的看代码吧）
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n,as[25],p,dp[25],qs[25];
vector<int> cs[25];
int main(){
	cin>>n;
	for(int i=1;i<=n;i++){
		int a;
		cin>>a;
		as[++p]=a;
		dp[p]=a;
	}
	for(int i=1;i<n;i++){//一共n-1行
		for(int j=i+1;j<=n;j++){
			int c;
			cin>>c;
			id(c)cs[j].push_back(i);
		}
	}
	for(int i=1;i<=n;i++){
		for(int j:cs[i]){
			if(dp[j]+as[i]>dp[i]){
				dp[i]=dp[j]+as[i];
				qs[i]=j;
			}
		}
	}
	int maxv=0;maxi=0;
	for(int i=1;i<=n;i++){
		if(dp[i]>maxv){
			maxv=dp[i];
			maxi=i;
		}
	}
	//以下是为了反序，题目要求输出从头到尾的路径
	int st[25]={},pp=0;
	while(qs[maxi]!=0){
		st[++pp]=maxi;
		maxi=qs[maxi];
	}
	st[++pp]=maxi;
	while(pp>=1){
		cout<<st[pp--]<<" ";
	}
	//输出结果（最大雷数）
	cout<<endl<<maxv;
}
```
