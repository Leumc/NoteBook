# 问题分析：

事实上，看出这道题是广度优先搜索是很简单的，我非常迅速地写出了如下代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n,m,ox,oy,ns[405][405];
int ds[9][2] = { {},{-1,2},{1,2},{-1,-2},{1,-2},{2,1},{2,-1},{-2,1},{-2,-1} };
void bfs(int x, int y,int cnt) {
	if ((x<1 || x>n || y<1 || y>m) || (ns[x][y] != -1 && cnt >= ns[x][y])) {
		return;
	}
	else {
		ns[x][y] = cnt;
		for (int i = 1; i <= 8; i++) {
			bfs(x + ds[i][0], y + ds[i][1], cnt + 1);
		}
	}
}
int main() {
	cin >> n >> m >> ox >> oy;
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= m; j++) {
			ns[i][j] = -1;
		}
	}
	bfs(ox, oy, 0);
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= m; j++) {
			cout << ns[i][j] << " ";
		}
		cout << endl;
	}
}
```

这段代码是可以得出正确的结果的，但评测时有一个测试点爆了TLE。看了题解之后发现几乎都是用队列解决的，并且都做了访问记录。

我本以为有了判断是否`cnt >= ns[x][y]`之后，不合理的分支会被剪枝，就不用记录了。但即便如此，**每层递归都会产生8个分支，尽管及时剪枝了，但这之间仍旧会有时间损失。当棋盘大小变大之后，这个时间损失会被指数级放大**（时间复杂度为$O(8^{m\times n})$），TLE在所难免（那个测试点的输入中，棋盘大小是长宽199的方阵）。而且由于`cnt >= ns[x][y]`这个判断必须得做，不做就会导致步数不是最小，所以我也没法去做访问记录……

所以我去学了一下题解区用的最多的这种BFS，连递归都省了。
# 用队列实现BFS：

先贴代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n, m, ox, oy, ns[405][405];
bool is_visit[405][405];
int ds[8][2] = { {-1,-2},{-1,2},{1,-2},{1,2},{2,-1},{2,1},{-2,-1},{-2,1} };
queue<pair<int, int>> q;
int main() {
	cin >> n >> m >> ox >> oy;
	memset(ns, -1, sizeof(ns));//-1(二进制全1)，可以用memset的
	ns[ox][oy] = 0; is_visit[ox][oy] = true; q.push(make_pair(ox, oy));
	while (!q.empty()) {
		int x = q.front().first,y=q.front().second;
		for (int i = 0; i < 8; i++) {
			int dx = x + ds[i][0], dy = y + ds[i][1];
			if (is_visit[dx][dy] || (dx<1 || dx>n || dy<1 || dy>m))continue;
			//越界或者已访问过，直接continue剪枝
			q.push(make_pair(dx, dy));
			is_visit[dx][dy]=true;
			ns[dx][dy]=ns[x][y]+1;
		}
		q.pop();//处理完后再出队
	}
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= m; j++) {
			cout << ns[i][j]<<" ";
		}
		cout << endl;
	}
}
```

## 首先一点是，用队列怎么保存坐标？

STL的队列是可以指定数据类型的，但**经典C的数组是不行的，因为数组没有独有的类型名**。这意味着我们似乎并不能指定一个定长的数组，用`vector`的话开销就过于大了。

写一个结构体自然是可以的，但其实STL不起眼的角落里，有一个十分完美的类型，那就是键值对，也就是`std::pair`。并不是只有写表的时候才能用键值对的，我们可以**用键来表示横坐标，值来表示纵坐标**，只要我们定义一个`queue<pair<int,int>>`对象，就能用队列来表示坐标了。
## 为什么这个写法可以表示最少的步数？

我们先看一下之前的写法为什么不做判断就不能得到最少步数，看这段代码：

```cpp
for (int i = 1; i <= 8; i++) {
	bfs(x + ds[i][0], y + ds[i][1], cnt + 1);
}
```

我的本意是在每一层直接把八种情况直接遍历出来，但实际上在遍历出第一种情况的时候，递归实际上就已经进入下一层了，后续都是这样，只有把在此基础上衍生出的所有可能情况都走到底之后才会继续第二种情况（感觉像是用广搜的思路写了一个深搜出来……）。这就会导致，**第一次到达某点可能是从最远的地方绕路过来的**。

而如果是真正意义上的广搜的话，就很容易证明：如果一个点被搜到多次，最早的一次一定是步数最少的一次。而**队列是一个FIFO的结构，如果把八种可能性逐一入队，那么它们一定先被处理**，这样才能实现真正意义上的广搜。

这样一来，我们就不必做那个`cnt >= ns[x][y]`的剪枝判断了，只需要判断是否访问过就行（即每个坐标最多只会入队一次）。布尔类型的二维数组占空间也不大，而时间复杂度能优化到$O(m\times n)$，至少不会指数爆炸了。
