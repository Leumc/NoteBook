# 问题分析：

本题有一个极为特殊的条件，即$max\{r_i-l_i\}\le min\{R_i-L_i\}$，这意味着，无论$l_i$与$r_i$取多少，线段的长度都会比任意一个$L_i$与$R_i$确定的“框“区间长度要短。而题目规定，区间”框“住线段的条件是有至少一半的线段处在这个区间内。

因此，我们可以得到一个充要条件：线段被”框“在区间内$\iff$这条线段的中点一定处在区间内

所以说，区间”框“住的线段条数$=$这个区间内的线段中点个数。我们并不需要关注线段的左右端点，我们只需要存储中点的位置即可。
### 中点不整？

在计算中点时，可能会出现这样一种情况，就是两个端点加起来除以二之后得到的结果是一个小数，如果直接用`/`运算符，小数部分会被省略，而在这个问题中小数肯定是不能被省略的，这会引起误差。

当然，如果真要存的话，用`map<double,int>`也不是不行，但这样做不仅比较麻烦，而且会占用更多的资源和时间。

实际上，只要我们将所有的值都乘上$2$，就能规避所有的小数问题。这道题的数据范围最大是$10^5$，乘上$2$之后就是最大$2\times 10^5$，这个范围我们还是可以接受的。
### 子区间问题——前缀和：

既然要求子区间内的某一个玩意的个数，那么肯定首选用前缀和来解。
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n,m,l,r,mds[2000010];//开两倍的数组
int main(){
	cin>>n>>m;
	for(int i=1;i<n;i++){
		cin>>l>>r;
		l*=2;r*=2;
		mds[l+r]++;
	}
	for(int i=1;i<=2000010;i++){
		mds[i]+=mds[i-1]
	}
	for(int i=1;i<=m;i++){
		cin>>l>>r;
		l*=2;r*=2;
		cout<<mds[r]-mds[l-1]<<endl;
	}
	return 0;
}
```
