# 问题分析：

国际象棋中，皇后是最强大的棋子，它可以攻击它所在的行列以及两条斜线上的棋子。本题要求我们找到这样一种情况：**在N行N列的棋盘上放置N个皇后，使得这N个皇后无法相互攻击，也就是所有的皇后必须放置在不同行、不同列以及不同斜线上。**

早在1848年，马克斯·贝瑟尔就提出了这个问题（标准的$8\times8$棋盘），但直到计算机被发明，这个问题才有了确切的解（共92个）。这是为什么？如果我们把棋盘改成$4\times4$，就可以很容易地得到唯一的两个解，穷举即可：

| ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  |
| :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: |
| \|  |     |  Q  |     |     | \|  |     |     |  Q  |     | \|  |
| \|  |     |     |     |  Q  | \|  |  Q  |     |     |     | \|  |
| \|  |  Q  |     |     |     | \|  |     |     |     |  Q  | \|  |
| \|  |     |     |  Q  |     | \|  |     |  Q  |     |     | \|  |
| ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  | ——  |
而当我们换到$8\times8$的棋盘上，如果用穷举法的话，尝试次数会达到$8^8=16,777,216$次，这对于人类来说显然是近乎不可能的。而对于这种重复、大量的计算工作，计算机表示最简单了。

当然，穷举法还是太不优雅了，我们可以进行一下优化。这里可以用一个经典的算法——回溯法，即发现当前路径是死路时，回到上一步重新选择。

基于DFS的回溯法核心模板如下：

```cpp
void dfs(参数){
	操作
	dfs(下一层参数);
	撤销操作
}
```

如果当前位置是死路，不会进行到下一层，那么本层执行完毕后就会回到上一层的dfs调用处，执行下一行代码，即撤销操作。
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n,ans[30],p,cnt;
bool cs[30],f1[50],f2[50];
void dfs(int r){
    if(r>n){//如果到达了n+1行，那证明n个皇后都已经放下了
        cnt++;
        if(cnt>3)return;//如果已经有了三个答案，就return，即不输出
        else{
            for(int i=1;i<=n;i++){
                cout<<ans[i]<<" ";
            }
            cout<<endl;
        }
        return;//避免递归继续深入
    }
    for(int i=1;i<=n;i++){
        if((!cs[i])&&(!f1[i+r])&&(!f2[r-i+n])){//判断当前位置是否可放置
            ans[++p]=i;//记录位置
            cs[i]=true;f1[r+i]=true;f2[r-i+n]=true;//标记占用
            //主对角线方向上的坐标，如果处在同一斜线，横纵坐标之和一致，即x+y
            //副对角线方向则是差值一定，加n使之为正，即x-y+n
            dfs(r+1);
            p--;//回退记录
            cs[i]=false;f1[r+i]=false;f2[r-i+n]=false;//取消标记占用
        }
    }
}
int main(){
    cin>>n;
    dfs(1);
    cout<<cnt;
    return 0;
}
```
