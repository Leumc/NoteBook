# 问题分析：

对于这道题，一眼就能看出是个图论问题，所以想到建立每个人的临接表。如果采用这种方法，在图的建立的时候十分顺利，但到了询问的时候，就会出现问题：如果我们想要判断两个人是否是关系，我们只能用深度/广度优先搜索把一个人庞大的关系网展开，尝试在其中寻找另一个人。**这种做法的时间复杂度直接爆炸，如果深度过深的话，甚至会出现栈溢出的风险！**

所以我们需要另一种方法……

回顾问题，我们真的需要这样一遍遍地去遍历一个人的所有亲戚吗？实际上是不需要的，**我们对它的路径并不关心**。而我们能够很容易地想到这样一个结论：如果两个人的祖先相同，那么这两个人一定是亲戚。

所以，如果我们定义`pt[x]`表示`x`的最老祖先，并且能够实时更新使之在一定规则下始终为当前已知信息下的祖先，那么我们就可以保证**当且仅当两个人的最老祖先一致时，两人之间有亲戚关系**。这样一来，我们的查询操作就很简单了。 ^741f50
# 并查集：

并查集，正如其名，一种主要功能是合并和查询的集合，通过选择集群中的一个元素作为全体代表，从而达到路径压缩的目的。

* 合并：如果我们指定两个人（`a`和`b`）之间应该有亲戚关系，那么我们看一下两个人对应的最老祖先（`pt[a]`和`pt[b]`），如果他们的最老祖先不一致，那么需要在他们所对应的两个亲戚集群之间建立亲戚关系。要怎么做呢？最简单的方法，~~*只要让两个祖先其中的一个认另一个当祖宗就行了！(doge)*~~
* 查询：[[「并查集」P1551 亲戚#^741f50|同样的逻辑]]，只要判断一下两个人的最老祖宗是否一致即可。
# 代码实现：

首先，在最开始，我们需要定义一个初始状态，如果一个人没有任何亲戚（最初始状态，没有任何信息），那么他的祖先是谁？**当然是他自己**！所以我们进行一下初始化：

```cpp
for(int i=1;i<=n;i++)pt[i]=i;
```

当然，我们也可以用一些C++的实用工具：

```cpp
std::iota(pt+1,pt+n,1);//从1到n,从1开始
```

初始化完成之后，我们需要实现寻找祖宗的函数，这是并查的基础，在这个过程中，我们可以一并实现路径的压缩：

```cpp
int find(int x){
	if(pt[x]==x)return x;//自己是自己的祖先，一定是终点，直接返回
	else{
		pt[x]=find(pt[x]);
		/*递归寻找祖先的祖先并更新，递归结束之后一定是最深层递归的数据。
		而最深层的祖宗也一定有祖宗，所以这个最深的祖宗的祖宗一定是他自己，
		从而触发边界判断*/
		return pt[x];
	}
}
```

然后我们可以实现一下合并函数，从而更方便于调用：

```cpp
void unite(int x,int y){
	int px=find(x),py=find(y);
	if(px!=py){
		pt[px]=py;
		//pt[py]=px也可以
	}
}
```
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int n,m,p;
int pt[5005];
int find(int x){
	if(pt[x]==x)return x;
	else{
		pt[x]=find(pt[x]);
		return pt[x];
	}
}
void unite(int x,int y){
	int px=find(x),py=find(y);
	if(px!=py)pt[px]=py;
}
int main(){
	cin>>n>>m>>p;
	for(int i=1;i<=n;i++)pt[i]=i;
	for(int i=1;i<=m;i++){
		int x,y;cin>>x>>y;
		unite(x,y);
	}
	for(int i=1;i<=p;i++){
		int x,y;cin>>x>>y;
		if(find(x)==find(y))cout<<"Yes"<<endl;
		else cout<<"No"<<endl;
	}
}
```
