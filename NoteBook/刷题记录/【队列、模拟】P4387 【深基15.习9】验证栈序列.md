# 问题分析：

这道题题面给的信息很少，似乎就在说基本常识一般，然而我还是没搞懂它说的到底是个什么意思。借助D老师神力，我来解释一下题目究竟是让我们干什么。

假设有一个栈和一个给定的序列，按照序列的顺序令所有元素入栈，你可以在**任意时刻令任意数量的元素出栈**（题目没规定所以可以），那么给出的验证序列是否是其中一种可能出栈序列？

例如，给定一个入栈序列`1,2,3,4,5`和一个出栈序列`1,3,4,2,5`，这个出栈序列是否是可能的？我们来试一下：
1. 首先将`1`压入栈中，然后立即将其弹出，现在的出栈序列是`1`，当前栈为空；
2. 然后令`2`入栈，不弹出任何元素，出栈序列不变，当前栈中为`2`；
3. 令`3`入栈，弹出`3`，出栈序列为`1,3`，当前栈中为`2`；
4. 令`4`入栈，并且令栈中所有元素弹出，出栈序列变为`1,3,4,2`，栈为空；
5. 令`5`入栈，然后立即弹出，出栈序列为`1,3,4,2,5`，栈为空，**验证成功**。

所以说，我们模拟的规则是：
* 在同一时间内只能且必须进行一次入栈操作；
* 在同一时间内可以进行任意次出栈操作（也可以什么都不做）。

每一次有一个元素入栈之后，如果栈顶元素与当前遍历到的出栈序列元素一致，就弹出栈顶元素（栈不为空的前提下），直到栈顶元素与出栈序列不匹配或者栈中所有元素都已经弹出。如果这一组测试用例的大循环结束后栈为空，那说明出栈序列是匹配的（**能够按照这个序列弹出所有元素**），输出`Yes`，否则输出`No`。

>另外，为了保证每组测试用例之间互不影响，需要在每组测试开始之前将栈清空。
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
int q, n, out[100005],in[100005];
stack<int> st;
int main() {
	cin >> q;
	for (int i = 1; i <= q; i++) {
		st = {};//清空栈
		cin >> n;
		for (int i = 1; i <= n; i++) {
			int x; cin >> x; in[i] = x;
		}
		for (int i = 1; i <= n; i++) {
			int x; cin >> x; out[i] = x;
		}
		int p = 1;
		for (int i = 1; i <= n; i++) {
			st.push(in[i]);
			while (!st.empty() && st.top() == out[p]) {
				st.pop();
				p++;
			}
		}
		if (st.empty())cout << "Yes" << endl;
		else cout << "No" << endl;
	}
}
```
