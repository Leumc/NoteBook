# 问题分析：

需要分析吗？标题上写的很清楚～需要使用Floyd（弗洛伊德）算法来寻找无向图上某两点之间的最短路径。
# Floyd算法概述（假设不存在负权环）：

**我先说一下什么是**`负权环`：如果一个路径上的权重和为负数，那么如果再重复一遍这个路径，总权重和会变得更小（负的更多）。此时就出现了一个问题，即走的越多权重和越小，权重和越小路径（我们认为的）越短，而我们要找的是最小短的路径，如果有负权环的话就会没完没了，**你永远无法找到你想要的最短路径，因为走得越多路径越短，所以路径越远路径越短（不是在搞抽象）。**

接下来正式介绍一下Floyd算法，Floyd算法是一种使用动态规划思想来寻找无向图上某两点之间最短路径的算法，**相对比较暴力，适用于规模不大的图**（比如节点数小于等于500左右）。

我们定义`dp[i][j]`为从节点$i$到节点$j$的最短距离，如何判断一个路径是否是对应的最短路径？我们可以将这个问题拆分为是否经过路径上某一个节点，对于节点$k$，有两种状态，即**选与不选**：

1. 如果不选，那么路径就是直接从$i$到$j$，权重和就是$dp[i][j]$；
2. 如果选的话，那么路径就是先从$i$到$k$，载从$k$到$j$，权重和为$dp[i][k]+dp[k][j]$

**所以状态转移方程就是：**
$$dp[i][j]=\min(dp[i][j],dp[i][k]+dp[k][j])$$
代码实现如下：

```cpp
for(int k=1;k<=n;k++){
	for(int i=1;i<=n;i++){
		for(int j=1;j<=n;j++){
			dp[i][j]=min(dp[i][j],dp[i][k]+dp[k][j]);
		}
	}
}
//这代码怎么看怎么暴力，三重for循环恐怖如斯
```

转移过程就是这样了，然后还有一个很重要的点，那就是**初始化**：

1. 一个节点到它自己的初始距离是零；
2. 两个节点之间如果联通，那么它们之间的初始距离是它们之间边的权重值；
3. 如果两个节点之间不联通，那么初始距离应该是一个极大值（**根据题目修改，确保相加不溢出**）
# 完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;
const int INF=1e9;//两倍是20亿，不会爆int
int n,m;
int dp[105][105];
int main(){
	cin>>n>>m;
	for(int i=1;i<=n;i++){
		for(int j=1;j<=n;j++){
			if(i==j)dp[i][j]=0;
			else dp[i][j]=INF;
		}
	}
	for(int i=1;i<m;i++){
		int u,v,w;cin>>u>>v>>w;
		dp[u][v]=w;dp[v][u]=w;//无向图，两个方向都要
	}
	for(int k=1;k<=n;k++){
		for(int i=1;i<=n;i++){
			for(int j=1;j<=n;j++){
				dp[i][j]=min(dp[i][j],dp[i][k]+dp[k][j]);
			}
		}
	}
	for(int i=1;i<=n;i++){
		for(int j=1;j<=n;j++){
			cout<<dp[i][j]<<" ";
		}
		cout<<endl;
	}
}
```
